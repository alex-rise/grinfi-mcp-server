# Grinfi MCP — Повний гайд

**Підключи Claude AI до Grinfi та керуй усім аутрічем звичайною мовою.**

Grinfi MCP — це міст між Claude і твоїм Grinfi-акаунтом. Ти пишеш Claude що треба зробити, а Claude виконує це в Grinfi: знаходить контакти, запускає автоматизації, пише листи, обробляє задачі й багато іншого. Без клікання по меню, без копіпасту даних між вкладками.

---

## Як підключитись

Є три способи — залежно від того, яким продуктом Claude ти користуєшся.

### Варіант 1 — Claude.ai (браузер, без налаштувань)

1. Відкрий [claude.ai](https://claude.ai) → **Settings → Integrations**
2. Натисни **Add Integration** і встав: `https://mcp.grinfi.io`
3. Claude попросить залогінитись через твій Grinfi email і пароль
4. Готово — Claude тепер має доступ до твого Grinfi-воркспейсу

### Варіант 2 — Claude Code (термінал)

Запусти у терміналі:

```bash
claude mcp add grinfi --transport http https://mcp.grinfi.io
```

Claude Code відкриє вкладку браузера для OAuth-авторизації. Після входу всі інструменти доступні в терміналі.

### Варіант 3 — Claude Desktop (локальна установка)

1. Скачай [репозиторій Grinfi MCP](https://github.com/alex-rise/grinfi-mcp-server) (зелена кнопка "Code" → "Download ZIP")
2. Отримай API-ключ у **Grinfi → Settings → API Keys**
3. Запусти інсталятор:
   - **macOS/Linux:** двічі клацни `install.sh` або виконай `./install.sh` у терміналі
   - **Windows:** двічі клацни `install.bat`
4. Встав API-ключ за запитом
5. Перезапусти Claude Desktop

> **Кілька команд:** Якщо ти керуєш кількома Grinfi-воркспейсами, задай змінну середовища `GRINFI_TEAM_KEYS` зі значеннями `teamId:apiKey` через кому. Використовуй `list_my_teams` і `switch_team` для перемикання між командами прямо в розмові.

## Що вміє Claude в Grinfi

Grinfi MCP надає Claude **158 інструментів**, організованих у 14 функціональних блоків. Нижче — практичний гайд по кожному.

---

### 1. Контакти

Знаходь, створюй, редагуй і керуй лідами. Claude може знайти контакт за LinkedIn-URL, email або ім'ям — і одразу показати посилання на профіль у Grinfi та LinkedIn.

**Інструменти:** `find_contact`, `get_contact`, `search_contacts`, `update_contact`, `delete_contact`, `upsert_contact`, `add_contact_to_automation`, `add_new_contact_to_automation`, `cancel_contact_from_automations`, `cancel_contact_from_all_automations`, `change_contact_pipeline_stage`

**Приклади запитів:**
- *«Знайди контакт з LinkedIn linkedin.com/in/john-smith і покажи його дані»*
- *«Покажи всіх контактів з компанії Stripe, з якими ми ще не контактували»*
- *«Онови посаду Марії Гарсіа на VP of Engineering»*
- *«Перемісти всіх контактів з тегом 'гарячий-лід' на стейдж 'Переговори'»*
- *«Додай Джона до автоматизації Q2 Outreach»*

## Приклади реальних сценаріїв

Це повноцінні багатокрокові воркфлоу, які можна описати Claude в одному повідомленні:

**Ранковий огляд:**
> *«Перевір непрочитані розмови, покажи задачі на сьогодні та які автоматизації мали відповіді за останні 24 години»*

---

**Запуск нової кампанії:**
> *«Клонуй автоматизацію 'SaaS Cold Outreach', перейменуй на 'Fintech Q2 2026', перемісти в папку 'Q2 Кампанії', потім додай усіх контактів зі списку 'Series B Fintech' до неї»*

---

**Дослідження відповіді:**
> *«Знайди контакт, який відповів на нашу кампанію сьогодні, покажи повну історію листування, дані компанії, поточний стейдж і нотатки на профілі»*

---

**Масове очищення:**
> *«Знайди всіх контактів зі списку 'Старі ліди 2024', з якими ми жодного разу не контактували, додай їм тег 'архів-2024', перемісти на стейдж 'Заархівовано' та видали їх із всіх активних автоматизацій»*

**Налаштування нового відправника:**
> *«Створи профіль відправника для Анни Мюллер, встанови робочі години пн–пт 8:00–17:00 за берлінським часом зі смарт-лімітами, потім підключи LinkedIn-браузер із німецьким проксі»*

## Поради для кращих результатів

- **Називай точно** — якщо є кілька автоматизацій зі схожими назвами, вказуй ключові слова для уточнення
- **Claude пам'ятає контекст** — у рамках однієї розмови Claude знає, які контакти, автоматизації або дані вже обговорювались
- **Проси підтвердження** для небезпечних дій — Claude запитає перед видаленням контактів або зупинкою активних автоматизацій
- **Комбінуй дії природно** — можна попросити знайти щось і одразу діяти на підставі результату в одному повідомленні
